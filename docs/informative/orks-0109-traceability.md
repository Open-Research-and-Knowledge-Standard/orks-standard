## Phase 2 Projection Fixture Plan

Phase 2 will place exact pristine projections, edited copies, expected outcome
records, and recomputed candidate revision vectors in `orks-conformance`.
Positive fixtures will cover byte-exact rendering for every MVP family,
unchanged round trips, valid human-edit successor candidates, protected map
code blocks and hierarchy, degraded warnings, exact bounds, every closed
outcome, dependency ordering, and a final atomic state recheck. Negative
fixtures will cover every byte and marker mutation, bundle self-inclusion,
identity mismatch, stale-edit heads, protected edits, malformed and hostile
edit bodies, every fence mutation, Markdown image/link/autolink/HTML/backtick
activation, injection, marker spoofing, duplicate and reordered controls,
conflicts, unsupported negotiation, resource refusal, diagnostic non-echo,
and absence of canonical mutation.

The executable suite will compare exact bytes, parsed origin tuples,
preflight stage, selected outcome, complete proposed six-member semantic value,
framed claimed revision identifier, candidate record, and before/after state.
At least two independent rendering and import implementations from different
language ecosystems will process the same fixture corpus without sharing a
projection parser or serializer. No implementation output becomes authority;
every expected value cites its governing rule and normative example.
